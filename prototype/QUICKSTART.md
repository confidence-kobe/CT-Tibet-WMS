# 🚀 原型快速开始指南

## 📖 原型演示说明

本原型系统是**西藏电信仓库管理系统**的交互式原型，包含PC管理端和微信小程序端，展示了完整的业务流程和用户交互。

## 🎯 快速访问

### 方式一：直接打开（推荐）
直接用浏览器打开 `prototype/index.html` 文件即可。

### 方式二：使用本地服务器
```bash
cd prototype
python -m http.server 8080
# 访问 http://localhost:8080
```

### 方式三：VS Code Live Server
1. 安装 VS Code 扩展：Live Server
2. 右键 `index.html` → "Open with Live Server"

## 📱 原型演示流程

### PC端演示流程

#### 流程1：仓库管理员日常操作
1. **登录系统**：使用 `wl_warehouse / 123456` 登录
2. **查看工作台**：查看今日数据、待办事项、库存预警
3. **处理待审批**：进入"审批管理" → 查看待审批申请 → 审批通过
4. **直接出库**：进入"出库管理" → 新建出库单 → 填写信息提交
5. **新建入库**：进入"入库管理" → 新建入库单 → 添加物资提交
6. **查看库存**：进入"库存查询" → 搜索物资 → 查看库存详情
7. **统计报表**：进入"统计报表" → 查看各类统计数据

#### 流程2：普通员工申请物资
1. **登录系统**：使用 `wl_user1 / 123456` 登录
2. **新建申请**：进入"我的申请" → 新建申请 → 填写用途和物资
3. **查看申请状态**：在"我的申请"中查看申请状态
4. **查看库存**：进入"库存查询" → 确认物资库存情况

### 小程序端演示流程

#### 仓管员端

## ❓ 常见问题

### Q1：为什么页面刷新后数据会重置？
A：这是静态原型，所有数据都在前端内存中，刷新后会重新加载初始数据。实际项目中数据会保存在数据库。

### Q2：能否真正提交数据？
A：不能。这是纯前端原型，所有操作都是模拟的，不会真正保存数据。

### Q3：小程序端为什么在电脑上显示？
A：这是小程序的网页原型，用于演示交互效果。实际小程序需要用微信开发者工具开发。

### Q4：原型支持哪些浏览器？
A：建议使用 Chrome、Edge、Firefox 等现代浏览器。不支持 IE。

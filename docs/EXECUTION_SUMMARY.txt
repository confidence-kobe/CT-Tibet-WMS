================================================================================
                                 问题概述
================================================================================

症状:
  登录API返回 HTTP 500 错误
  错误信息: "Unknown column 'deleted' in 'field list'"
  影响: 整个系统无法使用（无法登录）

根本原因:
  字段映射不一致 - 数据库使用 'is_deleted' 列，而Java代码期望 'deleted' 列

发现过程:
  1. 检查error message和stack trace
  2. 分析MyBatis-Plus生成的SQL语句
  3. 审查Java实体类定义 (BaseEntity.java)
  4. 检查MyBatis-Plus全局配置 (application.yml)
  5. 查询数据库表结构 (schema.sql)
  6. 发现三层不一致的证据

================================================================================
                              修复方案概览
================================================================================

方案选择: 方案A（推荐）- 数据库列改名

操作:
  将所有表的 'is_deleted' 列改名为 'deleted'

  影响的表:
  ✓ tb_user
  ✓ tb_role
  ✓ tb_dept
  ✓ tb_menu
  ✓ tb_material
  ✓ tb_warehouse

数据影响: 无（仅改列名，所有数据完全保留）

================================================================================
                          根本原因详细分析
================================================================================

问题的三层不一致:

1. Java实体层 (BaseEntity.java)
   - 字段名: deleted （驼峰式）
   - 注解: @TableLogic

2. MyBatis-Plus配置层 (application.yml)
   - 配置: logic-delete-field: deleted （指向Java字段名）

3. 数据库表结构层 (schema.sql)
   - 列名: is_deleted （蛇形式）

症状: 当MyBatis-Plus生成SQL时，使用Java字段名 'deleted' 作为列名
      但数据库中不存在 'deleted' 列，只有 'is_deleted' 列
      导致: "Unknown column 'deleted'" 错误

解决: 将数据库列名改为 'deleted'，完全对齐三层配置

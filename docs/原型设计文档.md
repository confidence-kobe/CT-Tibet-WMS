## 1. 设计概述

### 1.1 设计目标

- **PC端**：面向仓管员、部门管理员、系统管理员，提供完整的管理功能
- **小程序端**：面向仓管员、普通员工，提供移动便捷操作

### 1.2 核心用户场景

| 用户角色 | 核心场景 | 主要平台 |
|---------|---------|---------|
| 系统管理员 | 用户管理、系统配置 | PC端 |
| 部门管理员 | 部门数据管理、查看报表 | PC端 |
| 仓库管理员 | 出入库操作、审批申请 | PC端 + 小程序 |
| 普通员工 | 提交申请、查看库存 | 小程序 |

### 1.3 设计原则

1. **简洁高效**：减少操作步骤，提高工作效率
2. **角色适配**：不同角色看到不同的菜单和功能
3. **状态清晰**：所有业务状态一目了然
4. **响应式**：PC端支持不同分辨率
5. **易用性**：符合用户操作习惯

## 5. 交互流程设计

### 5.1 员工申请领用完整流程

```mermaid
graph TD
    A[员工打开小程序] --> B[点击新建申请]
    B --> C[填写用途]
    C --> D[选择物资]
    D --> E[输入数量]
    E --> F{继续添加?}
    F -->|是| D
    F -->|否| G[提交申请]
    G --> H[显示提交成功]
    H --> I[发送通知给仓管]

    I --> J[仓管收到通知]
    J --> K[打开审批页面]
    K --> L[查看申请详情]
    L --> M{是否通过?}

    M -->|通过| N[填写审批意见]
    N --> O[点击通过]
    O --> P[自动创建出库单]
    P --> Q[状态:待领取]
    Q --> R[发送通知给员工]

    M -->|拒绝| S[填写拒绝原因]
    S --> T[点击拒绝]
    T --> U[更新申请状态]
    U --> V[发送通知给员工]

    R --> W[员工到仓库领取]
    W --> X[仓管确认出库]
    X --> Y[扣减库存]
    Y --> Z[完成]
```

### 5.2 仓管直接出库流程

```mermaid
graph TD
    A[仓管登录系统] --> B[点击新建出库单]
    B --> C[选择领用人]
    C --> D[填写用途]
    D --> E[选择物资]
    E --> F[输入数量]
    F --> G{库存充足?}

    G -->|否| H[提示库存不足]
    H --> F

    G -->|是| I{继续添加?}
    I -->|是| E
    I -->|否| J[提交出库单]

    J --> K[生成出库单号]
    K --> L[立即扣减库存]
    L --> M[状态:已出库]
    M --> N[完成]
```

### 5.3 审批超时处理流程

```mermaid
graph TD
    A[定时任务执行] --> B[查询待审批申请]
    B --> C{超过24小时?}

    C -->|是| D[发送催办通知]
    D --> E[通知仓管员]

    C -->|否| F[继续监控]

    A --> G[查询待领取出库单]
    G --> H{超过7天?}

    H -->|是| I[自动取消出库单]
    I --> J[更新申请单状态]
    J --> K[发送超时通知]
    K --> L[通知员工]

    H -->|否| M[继续监控]
```

## 8. 响应式设计

### 8.1 PC端断点

```
超大屏幕: ≥ 1920px  显示完整布局 + 大间距
大屏幕:   ≥ 1440px  显示完整布局
中屏幕:   ≥ 1024px  正常布局
小屏幕:   < 1024px  收起侧边栏,使用顶部菜单
```

### 8.2 表格响应式

```
宽屏: 显示所有列
窄屏: 隐藏次要列,保留核心信息 + 操作列
移动: 卡片式布局,每行显示一条记录
```
